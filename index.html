“互联网+”模式下的剧院改革

##背景与现状

　　在当前云计算和大数据的环境下，任何企业，任何产品，任何服务可能都面临着一个在互联网时代怎样进一步发展的问题。营销方式的改变，战略防线的选择，商业模式的变化，都绕不开“互联网化”。
　　现在的电影院行业已经实现了卖票互联网化了，观众在进影院之前已经把票在手机上买好，还选好了位子，到了电影院，取票机前输个号就可以进场看电影了。而这只是互联网模式下的一个例子。

##互联网化
　　传统行业的“互联网化”通常有一个非常简单的框架，互联网化就三步。剧院的改革也不例外。
　　1.在线。
　　首先做一个剧院的在线售票网站，在这个售票网站上，客户可以浏览目前在售的儿童剧，木偶剧，皮影等演出详情，同时可以在线选座位。等功能。

　　2.互动。
　　以前的传媒技术，报纸，广播，电视，核心技术都是单向传播的。互联网是一次大的技术革命，所有的交流都是双向的，而且是实时的，不是滞后的。
　　所以，在看演出的过程中，或是看演出结束后，注册用户在网站的评论功能模块的支持下发表评论，及时的进行反馈。互动可以让商家实实在在的接触到用户，并对他们进行服务。
　　3.联网。
　　互联网会将越来越多的点连在一起，可以便于跟类似或者同样的企业产生更多的协同和利益。创造交叉组合模式，产生新的利益。

##产品构想与大致模块
　　1.第一阶段。
　　剧院的在线售票网站及微信公众号在线购票。其中包括以下模块。
　　1⃣️网站信息新闻cms模块。
　　2⃣️用户评论模块。
　　3⃣️后台管理模块。(便于管理用户的数据情况)
　　4⃣️财会系统模块。(依靠运算处理，便于统计每一个场次的售票情况，上座率，盈利金额以及每一个注册用户在一定时间内的观看场次及消费金额。)
　　5⃣️在线选座模块。(涉及并发处理)
　　6⃣️在线支付模块。(支付宝，微信支付)
　　7⃣️短信通知模块。(用户注册，交易成功后短信通知)
　　8⃣️出票。(爱普生TM-T86L打印机读写操作)
　　9⃣️会员管理模块。(会员包括家庭会员和单人会员可以办理年卡，及月卡，享受优惠。会员区别于普通注册用户是，在注册会员是，需要上传照片，便于出票时现场人员的查询校验。)
　　🔟微信公众号手机购票模块。
　　2.第二阶段。
　　1⃣️对接国内流量比较大的互联网售票系统(美团，格瓦拉生活，大麦票务等)
　　3.第三阶段。
　　1⃣️定制化推荐系统。(大数据分析系统可以挖掘用户的兴趣点，向注册用户定制化的推送推荐信息，如果剧场在后期的发展设想中除了表演外，还会涉及到一些其他的衍生行业，比如木偶的销售之类的，也可推荐用户去购买。)
　　2⃣️聚合观众会员人群，探讨文化衍生品的社区电商化的可能性。
　　3⃣️探讨在线直播的可能性。 　
　　4.第四阶段。
　　1⃣️在一定规模用户后，开发ios/android的剧场app。

##项目架构

###技术选型
　　开发语言：使用python作为server端的主要开发语言。python因其自由，简单，易用，第三方库强大而著称。
　　web框架：使用flask来作为web框架。flask是一个很好的支持webapi开发的python web框架。可以快速地迭代式开发。
　　开发环境：linux os 　

###部署架构　　
　　Nginx：前端代理服务器及负载均衡器。可以直接代理访问静态html提高server性能。
　　Gunicorn：一个开源Python WSGI UNIX的HTTP服务器，速度快（配置快、运行快）、简单，默认是同步工作，支持Gevent、Eventlet异步。可以在Django项目中修改worker工作方式为gevent来部署异步服务器框架提高IO性能。
　　supervisor：linux下的守护进程。来管理维护gunicorn进程。
　　DB：分布式数据库postgresql。方便以后分布扩展。
　　

###上线环境
　　server平台：阿里云(程序运行环境)
　　流媒体平台：七牛云(图片，视频等流媒体)

flask.exthook.ExtDeprecationWarning警告的消除

flask升级到0.11版后，弃用了以 flask.ext.xxx 导入扩展模块的形式，改为 flask_xxx。
如果仍然沿用原来的形式，flask会报警告flask.exthook.ExtDeprecationWarning。

flask团队提供了flask-ext-migrate的转换工具。

pip安装： 

$ pip install flask-ext-migrate

转换： 

$ flask_ext_migrate xxx.py

但目前pip上的版本上存在bug，最新的github上fix了这个问题。
这个转换工具一次只能转换一个python文件。
随便写个shell批量处理吧。 

$ for f in `find . -name "*.py"`
> {
> flask_ext_migrate $f
> }

mac下批量替换文本

在mac下使用sed与linux下稍微有一些不同。
-i 参数可以指定备份源文件名 

sed -i "bk" "s/Cat/Dog/g" example.txt

替换example.txt文件中的Cat->Dog时，会生成备份文件example.txtbk。也可以指定不生成备份文件，-i参数为“”。

批量替换命令如下： 

sed -i "" "s/Cat/Dog/g" `grep Cat -rl ./`

用grep查找出当前文件夹下含有Cat的文件，然后替换成Dog。并且不指定备份文件。

pip freeze

在本地python开发时，可以生成requirements.txt文件，用于记录所有依赖包及其精确的版本号。 

pip freeze >requirements.txt

可创建与本地完全一致的副本环境。 

pip install -r requirements.txt

使用virtualenv

虚拟环境非常有用。可以在系统的python解释器中避免包的混乱和版本的冲突。

安装virtualenv

sudo pip install virtualenv

创建python虚拟环境

在工程文件夹下创建python虚拟环境。创建虚拟环境后，当前文件夹中会出现一个子文件夹，名字为下述命令中指定的参数venv。 

virtualenv venv

激活

source venv/bin/activate

退出

deactivate

mac下搭建hexo至github

hexo是一款基于Node.js的静态博客框架:hexo github

安装nodejs

brew install node

安装hexo

npm install -g hexo
hexo init <folder>
npm install

hexo语法

hexo new "postName" #新建文章
hexo generate #生成静态页面至public目录
hexo server #本地预览，默认4000端口
hexo deploy ＃发布

配置

修改hexo根目录下的_config.yml文件,xxx为github的账户名称。 

deploy:
  type: git
  repository: https://github.com/xxx/xxx.github.io.git
  branch: master

在github创建xxx.github.io

安装主题

个人比较喜欢yilia github

发布

$ hexo d -g
